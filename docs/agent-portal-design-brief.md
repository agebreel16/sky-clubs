# Sky Clubs — Agent Portal · Design Brief
> لـ Claude Design — تصميم واجهة بوابة الوكيل الذاتية

---

## 1. رؤية المشروع

بوابة ذاتية يفتحها الوكيل عبر رابط آمن يُشاركه معه المدير — بدون كلمة مرور.  
الهدف ليس مجرد عرض أرقام، بل **تجربة تُشعر الوكيل بأنه داخل غرفة تحكم حية** تعكس أداءه لحظةً بلحظة.  
الواجهة تُحفّز، تُنبّه، وتُشعره بالتقدم نحو هدفه.

**الكلمة المفتاحية:** مباشر · حي · تحفيزي · فخم

---

## 2. Design System

### 2.1 الألوان

```
Primary Blue    #0ea5e9   ← أزرق سماوي (اللون الرئيسي للعلامة)
Primary Dark    #0284c7   ← تدرج أغمق للـ hover
Success Green   #10b981   ← ترقية / مدفوع / إيجابي
Warning Amber   #f59e0b   ← تحذير / معلق
Danger Red      #ef4444   ← تهبيط / خسارة / خطر
Purple          #8b5cf6   ← خطوط جديدة / إنجازات
Slate Dark      #1e293b   ← نصوص رئيسية
Slate Mid       #64748b   ← نصوص ثانوية
Slate Light     #94a3b8   ← placeholders / timestamps
Surface         #f1f5f9   ← خلفية الصفحة
Card            #ffffff   ← خلفية البطاقات
Border          #e2e8f0   ← حدود خفيفة

Gradient Hero   linear-gradient(135deg, #0ea5e9 0%, #6366f1 100%)
Gradient Warm   linear-gradient(135deg, #f59e0b 0%, #ef4444 100%)
Gradient Green  linear-gradient(135deg, #10b981 0%, #0ea5e9 100%)
```

### 2.2 التايبوغرافي

```
Font Family : 'Alexandria', sans-serif (Google Fonts)
Direction   : RTL — العربية فقط
Sizes       : 11px timestamps · 12px badges · 13px secondary · 14px body · 16px section titles · 20px card titles · 28-40px KPI values
Weights     : 300 thin · 400 regular · 600 semibold · 700 bold
```

### 2.3 الحركة والانيميشن

```
Micro-animations : 200-300ms ease-out (hover, click, state change)
Page transitions : 400ms fade + translateY(8px) عند تبديل الصفحات
Number counters  : count-up animation عند ظهور KPI cards (0 → القيمة الحقيقية)
Progress bars    : تملأ من 0% → القيمة الحقيقية عند ظهور الصفحة
Chart draw       : line draws from left to right (1.2s ease-out)
Skeleton loading : shimmer effect قبل تحميل البيانات
Pulse            : نبضة خفيفة على الأرقام عند تحديثها بالـ polling
```

### 2.4 المكونات المشتركة

```
Border Radius : cards 16px · buttons 8px · badges 20px · toasts 14px
Shadow        : card → 0 1px 3px rgba(0,0,0,0.07), 0 4px 16px rgba(0,0,0,0.04)
Shadow hover  : 0 8px 32px rgba(14,165,233,0.15)
Border left/right: 4px solid [color] للبطاقات ذات الأولوية
```

## 3. Shell — الهيكل الخارجي

### 3.1 Navbar (أعلى ثابت)

```
┌─────────────────────────────────────────────────────────┐
│  🏆 Sky Clubs        [اسم الوكيل] [نادي القمة ▾] 🔔² خروج │
└─────────────────────────────────────────────────────────┘
```

- **خلفية:** gradient أزرق (Primary Blue → Primary Dark)
- **الشعار:** أبيض · font-weight 700 · مع تأثير glow خفيف
- **اسم الوكيل:** أبيض · مع أول حرف في avatar دائري ملوّن
- **Badge النادي:** pill شفاف أبيض (border 1px rgba(255,255,255,0.3)) · يضيء عند hover
- **جرس الإشعارات:** أيقونة SVG أبيض · عداد أحمر أعلاه · عند وصول إشعار جديد: يهتز (wiggle animation 600ms) + يتضاعف العداد بـ scale pop
- **زر خروج:** border خفيف · hover → خلفية بيضاء شفافة 10%

### 3.2 Navigation — Desktop (تبويبات أفقية تحت الـ navbar)

```
┌──────────────────────────────────────────────────┐
│  الرئيسية │ الأداء │ الإشعارات 🔴² │ المكافآت │ اليانصيب │ السجل  │
└──────────────────────────────────────────────────┘
```

- **خلفية:** أبيض · حد سفلي 1px slate
- **التبويب النشط:** خط سفلي 2px أزرق + لون أزرق + خلفية زرقاء شفافة خفيفة
- **Hover:** انتقال سلس 200ms
- **"الإشعارات":** يُظهر عداد الغير مقروء كـ badge أحمر صغير
- **انتقال بين الصفحات:** fade + translateY(-8px → 0) 300ms

### 3.3 Navigation — Mobile (Bottom Bar)

```
┌─────────────────────────────────────────────────┐
│  🏠      📊      🔔²      💰      🎟️      📋    │
│ الرئيسية الأداء الإشعارات المكافآت اليانصيب  السجل │
└─────────────────────────────────────────────────┘
```

- ثابت أسفل الشاشة · خلفية بيضاء · shadow أعلاه
- الأيقونة النشطة: أزرق + scale 1.2 + dot أسفلها
- Safe area padding للـ iPhone notch
- التبويب النشط يُحرّك الأيقونة للأعلى 4px (bounce)

- **Animation:** وميض خفيف (pulse 2s infinite) على الـ border لجذب الانتباه

### صفحة 3 — الإشعارات (Notifications)

> ليست مجرد قائمة — بل مركز رسائل مُصنّف وجميل

---

#### 4.3.A — Header مع فلتر

```
┌─────────────────────────────────────────────────┐
│  🔔 الإشعارات  (3 غير مقروءة)                   │
│  ─────────────────────────────────────────────  │
│  [الكل ✓] [غير مقروءة 🔴3] [ترقية] [تحذير]        │
│                          [علّم الكل مقروءاً]     │
└─────────────────────────────────────────────────┘
```

- فلتر chips أفقية (scrollable على موبايل)
- "علّم الكل مقروءاً": زر ثانوي رمادي → أخضر عند click مع checkmark animation
- عداد الغير مقروء: badge أحمر يتغير بـ animation عند التغيير

---

#### 4.3.B — قائمة الإشعارات

كل إشعار:
```
┌───────────────────────────────────────────────────┐
│  🏆  مبروك الترقية!                        12:30  │
│      تهانينا! انضممت إلى نادي الألماس             │
│      ━━━━━━━━━━━━━━━━━━━━━━━━━━━━          │
│      [منذ 5 دقائق]                    [✓ مقروء]  │
└───────────────────────────────────────────────────┘
```

- **الغير مقروء:** خلفية زرقاء شفافة (rgba(14,165,233,0.05)) + نقطة زرقاء يسار البطاقة + عنوان bold
- **المقروء:** خلفية بيضاء عادية + خط أرفع
- **أنواع الإشعارات بألوان مختلفة:**
  - 🏆 ترقية → border أخضر
  - ⬇️ تهبيط → border أحمر
  - ⚠️ تحذير → border أصفر
  - ⭐ إنجاز → border بنفسجي
  - 🔔 عام → border أزرق
- **Animation:** عند mark as read → خلفية تتلاشى من أزرق إلى أبيض (transition 400ms)
- **Swipe to read** على موبايل (اختياري — تحسين UX)

---

#### 4.3.C — Toast System (يعمل على كل صفحات البوابة)

```
╔═══════════════════════════════════════╗
║  🏆                   [×]            ║
║  مبروك الترقية!                       ║
║  انضممت إلى نادي الألماس بنجاح        ║
║  ████████████████████░░░░░░░░░  8s   ║
╚═══════════════════════════════════════╝
```

**المواصفات التفصيلية:**
- **الظهور:** من أعلى يسار الشاشة (لـ RTL) — translateY(-100%) → translateY(0) + opacity 0→1 مع spring bounce خفيف
- **الشكل:** بطاقة بيضاء · border-radius 14px · shadow ثقيل متعدد الطبقات
- **الحد الجانبي:** 4px يمين · لون حسب النوع
- **الأيقونة:** في دائرة ملوّنة 40px (لون النوع بشفافية 15%)
- **Progress bar:** يمتد أسفل البطاقة · ينقص من 100% → 0% في 8 ثوانٍ · linear
- **الصوت:** 3 نغمات موسيقية متصاعدة (Web Audio API: 880→1100→1320 Hz) عند كل إشعار جديد
- **الضغط على البطاقة:** يعتبر الإشعار مقروءاً + يغلق البطاقة (fadeOut + translateY(-14px) + scale(0.96))
- **تراكم الإشعارات:** stacked cards (آخرها فوق الأول) — max 3 toasts في آن واحد، الرابع يدفع الأول للاختفاء
- **زر X:** يغلق فوراً بدون markRead
- **Auto-dismiss:** بعد 8 ثوانٍ (countdown واضح في progress bar)

**تأثير الجرس عند وصول إشعار:**
- wiggle animation: rotate(-15deg) → rotate(15deg) → rotate(-10deg) → 0 · 600ms
- العداد: scale(1.4) → scale(1) مع لون أحمر يومض

  │ 🎁 مكافأة         ████░░░░░░░   5 تذكرة │
  └─────────────────────────────────────────┘

  🏆 نادي الذهب
  ┌─────────────────────────────────────────┐
  │ 🎟️ دخول النادي    █████░░░░░░  8 تذكرة  │
  └─────────────────────────────────────────┘
```

- **لكل نوع:** progress bar بلون مختلف يُظهر نسبته من إجمالي تذاكره
- **أنيميشن:** الـ bars تملأ عند scroll إلى القسم
- **Hover على badge العدد:** tooltip "هذه X تذاكر من أصل Y لنادي [اسم النادي]"

**إمكانية إبداعية مقترحة:**
- Donut chart صغير يُظهر توزيع التذاكر بين الأندية بألوان مختلفة
- بطاقة "Jackpot potential" تُظهر: "كل 10 تذاكر = فرصة سحب"

│         نسبة التحويل انخفضت عن 50%

◉ ─── ⬆️ ترقية إلى نادي الذهب            2026-04-15
│         من نادي الفضة → نادي الذهب
│
○ ─── ⬇️ تهبيط من نادي الذهب            2026-03-20
│         انتهى عداد التهبيط
│
◉ ─── 🌟 إنجاز: أول وصول لنادي الذهب   2026-03-01
```

**التصميم:**
- **خط رأسي:** 2px رمادي (#e2e8f0) يمتد عبر كل الـ timeline
- **دوائر الخط:** ◉ للأحداث الإيجابية (مملوءة بلون النوع) · ○ للسلبية (حدود فقط)
- **البطاقة:** للجهة اليسرى من الخط (RTL: اليمين) مع shadow خفيف وحد جانبي ملوّن
- **Animation:** كل بطاقة تنزلق من اليمين عند scroll (Intersection Observer)
- **Badge النوع:** ملوّن حسب الحدث
- **التاريخ:** أعلى اليمين · رمادي فاتح
- **"السبب":** نص صغير رمادي تحت العنوان

**Scroll behavior:** عند فتح الصفحة → scroll تلقائي لآخر حدث (الأحدث في الأعلى)

## 5. نظام الإشعارات الفاخر — تفاصيل شاملة

### 5.1 دورة حياة الإشعار

```
[قاعدة البيانات] → [Livewire poll كل 5s] → [dispatch event] → [Alpine.js toast] + [صوت]
```

### 5.2 مستويات الإشعارات

| النوع | الأيقونة | اللون | الصوت | مدة الـ Toast |
|---|---|---|---|---|
| ترقية (promotion) | 🏆 | أخضر #10b981 | 3 نغمات صاعدة 🎵 | 10 ثوانٍ |
| تهبيط (demotion) | 📉 | أحمر #ef4444 | نغمة منخفضة واحدة | 12 ثوانٍ |
| تحذير (warning) | ⚠️ | أصفر #f59e0b | نغمتان متكررتان | 10 ثوانٍ |
| إنجاز (achievement) | 🌟 | بنفسجي #8b5cf6 | 3 نغمات + sparkle | 10 ثوانٍ |
| عام (info) | 🔔 | أزرق #0ea5e9 | نغمة واحدة هادئة | 8 ثوانٍ |

### 5.3 تصميم Toast Card

```
╔══════════════════════════════════════════════╗  ←  border-right 4px [color]
║                                        [×]  ║
║  ┌────────┐                                  ║
║  │   🏆   │   مبروك الترقية!                 ║  ← title 14px bold
║  │  bg10% │   انضممت إلى نادي الألماس بنجاح  ║  ← body 12px slate
║  └────────┘   منذ لحظات                     ║  ← timestamp 11px
║                                              ║
║  ██████████████████████████░░░░░░░  progress ║  ← 3px progress bar
╚══════════════════════════════════════════════╝
```

### 5.4 Stacking Behavior

```
أول toast → يظهر
ثاني toast → يدفع الأول للأسفل 4px + يضيف هو
ثالث toast → نفس الآلية
رابع toast → الأول يختفي تلقائياً ويحل محله الرابع
```

### 5.5 الحالة "قراءة كل الإشعارات"

```
قبل: 🔴 3 إشعارات جديدة
                    ↓ (click "علّم الكل مقروءاً")
بعد: ✅ (check animation) → عداد الجرس يتلاشى
```

## 6. العناصر التفاعلية الحية

> هذه العناصر تجعل الواجهة "تتنفس"

### 6.1 Micro-interactions

| العنصر | التأثير |
|---|---|
| KPI card hover | scale(1.03) + shadow أزرق |
| زر مقروء (click) | checkmark animation: ✓ يرتسم |
| Progress bar | تملأ من 0% عند أول ظهور (500ms ease-out) |
| Count-up numbers | 0 → القيمة الحقيقية (800ms) |
| Badge النادي في navbar | shimmer animation خفيف عند تحديثه |
| Timeline items | slide-in من اليمين عند scroll |

### 6.2 Polling Indicators (مؤشرات الحياة)

- **نقطة خضراء نابضة** (●) بجانب "الرئيسية" أو في footer تُشير إلى "الاتصال حي"
- عند poll ناجح → النقطة تومض وميضاً خفيفاً
- عند وصول إشعار → الجرس يهتز + النقطة تتحول أزرق لثانية

### 6.3 Skeleton Loading

```
قبل تحميل البيانات:
┌──────────────────────────┐
│ ░░░░░░░░░░░░░░░░░░░░░░░ │  ← shimmer gray bars
│ ░░░░░░ ░░░░░░░░░░░░░░░░ │
│ ░░░░░░░░ ░░░░░░░░░░░░░░ │
└──────────────────────────┘
```
- shimmer: عرض 200% يتحرك من اليمين لليسار (1.5s infinite)
- يختفي مع fade عند وصول البيانات

## 10. الخلاصة الإبداعية

الهدف النهائي: الوكيل يفتح البوابة فيرى **رقمه ينمو، جرسه يرن، وتذاكره تتراكم**.  
الواجهة ليست "صفحة عرض بيانات" — بل **لوحة قيادة تحفيزية حية** تشعره أن كل خط يبيعه يُضيف لرحلته.

- الأرقام تعدّ نفسها عند كل فتح
- الـ Progress bars تملأ بشكل مرئي
- الإشعارات تطرق بصوت وحركة
- Timeline يحكي قصته من البداية حتى الآن
- كل صفحة تُكمل الأخرى — تجربة واحدة متصلة
